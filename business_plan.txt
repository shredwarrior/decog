PROBLEM
-------
- 60% of hiring managers say graduates lack critical thinking skills.
- 40% of college students cannot identify an essay thesis.
- No accessible tool exists that scores argument quality with a transparent,
  calibrated framework. Current options are thin LLM wrappers that list
  fallacies without scoring, context, or educational value.
- Online discourse rewards rhetoric over reasoning. Users have no way to
  objectively assess whether their argument is structurally sound before
  publishing it.


POSSIBLE MARKETS
----------------
1. Students (university, debate clubs, writing courses)
   - Use case: self-check essays and arguments before submission.
   - Entry: free tier with basic analysis; professors assign it.

2. Content creators / bloggers / journalists
   - Use case: verify reasoning quality before publishing.
   - Entry: freemium with share badges for credibility signaling.

3. Online debaters / Reddit / X power users
   - Use case: shareable score cards that validate argument strength.
   - Entry: free tier with social sharing. Viral loop potential.

4. Corporate training / L&D departments
   - Use case: critical thinking assessment for employees.
   - Entry: institutional licensing (later stage, requires traction proof).

5. EdTech integration
   - Use case: plugin or API for LMS platforms (Canvas, Moodle).
   - Entry: API tier with per-analysis pricing.

Our edge:
- Only tool with calibrated scoring (grid-search optimized weights).
- Two-stage metadata pipeline (structural + epistemic) before judgment.
- Bias hierarchy with specificity-first deduplication.
- Philosophical razors (6) with pass/fail and falsifiability gating.
- Domain expertise: built by a cognitive psychologist, not a developer
  prompt-engineering a wrapper.
- Content moat: Cognition Today literature for premium explanations.

IMMEDIATE NEXT STEPS (post-MVP ship)
-------------------------------------
1. Ship to Railway with PostgreSQL.

2. Expert validation system:
   Add per-bias/fallacy/distortion rating buttons visible to admin users
   (you + invited experts). Options: "correct" / "loose" / "wrong".
   Store in a validations table: analysis_id, issue_key, issue_type,
   expert_id, rating, timestamp.
   This becomes ground truth for future model training AND a quality
   metric to advertise ("92% expert-validated accuracy").

3. Full metadata persistence in database:
   Expand analyses table to store: raw per-claim metadata (JSON blob),
   all detected issues with confidence scores, razor pass/fail with
   reasons, score breakdown, improvements, executive summary.
   One row per analysis, no additional LLM call needed to reproduce
   the full report. This is the training dataset for the future
   custom model.

4. Curate additional biases:
   Aditya to review taxonomy and add missing biases/distortions.
   System is designed for this — just add entries to definitions.json
   with name, description, penalty, and improvement hints.

5. Collect user feedback:
   Thumbs up/down on each analysis. Simple signal, high value over time.

6. Organic growth:
   Embed tool on Cognition Today. Share interesting analyses on social.
   Target psychology/philosophy subreddits and edu Twitter.
